Warum läuft der google index nicht über mein githib repo? https://github.com/mrtwiste/freifunk

Dass ein GitHub-Repository nicht (oder noch nicht) im Google-Index erscheint, hat in den meisten Fällen ein paar typische Gründe:
1. Fehlende externe Links (Backlinks)

Google entdeckt neue Seiten und Repositories primär dadurch, dass andere bereits bekannte Websites auf sie verlinken. Wenn keine oder nur sehr wenige externe Webseiten (z. B. Blogs, Foren, soziale Medien oder deine persönliche Website) direkt auf dein Repository verlinken, erfährt der Google-Bot oft gar nicht erst von dessen Existenz.
2. Niedrige Priorität im "Crawl Budget"

GitHub ist eine gigantische Plattform mit Millionen von Seiten. Google teilt der Domain ein gewisses Crawling-Budget zu und priorisiert Repositories, die viel Traffic, viele Stars, Forks oder regelmäßige Aktivitäten aufweisen. Neue oder kleinere Repositories landen in der Warteschlange weiter hinten. Es kann daher einige Wochen oder Monate dauern, bis Google ein Repo spontan erfasst.
3. Es war früher auf privat gestellt

Falls das Repository in der Vergangenheit kurzzeitig oder dauerhaft privat war, wird es von Google sofort de-indexiert. Nach dem Umstellen auf Public muss der Google-Bot die Seite erst wieder neu entdecken.
4. Dünner Inhalt / Fehlende Meta-Informationen

Wenn ein Repository wenig Text enthält (z. B. keine oder nur eine sehr kurze README.md sowie keine Repository-Beschreibung), bietet es Suchmaschinen kaum relevante Schlüsselwörter.
Was du tun kannst, um das Indexieren zu beschleunigen:  

    Ausführliche README.md hinzufügen: Schreibe eine detaillierte Dokumentation mit relevanten Begriffen rund um Freifunk und deine Skripte.  

    Repository-Details ausfüllen: Füge in den Repository-Einstellungen eine kurze Description sowie passende Topics/Tags hinzu.

    Verlinkungen setzen: Verlinke das Repository von anderen Webseiten aus (z. B. auf deiner Vereins-/Projekt-Website, in Foren oder in sozialen Netzwerken).  

    Auf GitHub Pages setzen (Optional): Falls du eine gh-pages-Website daraus erstellst, kannst du diese eigenständig bei der Google Search Console einreichen, was die Indexierung enorm beschleunigt.

## ssh-copy-id auf OpenWrt/UniFi: Problem und Lösung


`ssh-copy-id` kopiert SSH-Schlüssel standardmäßig nach `~/.ssh/authorized_keys`.

Auf OpenWrt/UniFi-Systemen werden diese jedoch oft in `/etc/dropbear/authorized_keys` benötigt.


**Problem:**


`ssh-copy-id` erkennt zwar OpenWrt, prüft aber (versionsabhängig) auch, ob der Benutzer "root" ist.

Auf UniFi-Geräten ist der Standardbenutzer jedoch "ubnt" (mit UID 0), was zu Problemen führt.


**Lösung:**


`ssh-copy-id` muss zusätzlich zur Benutzernamenprüfung auch die UID prüfen.

Ergänze dazu folgende Bedingung in der entsprechenden Zeile in `ssh-copy-id`:


`

[ -f /etc/openwrt_release ] && ([ "\$LOGNAME" = "root" ] || [ "\$(id -u)" = "0" ]) &&`


***Alternativen:***


Eigene Version von ssh-copy-id mit angepasster Prüfung erstellen und im Script auf diese verweisen.


Schlüssel manuell kopieren: scp id_rsa.pub ubnt@<gerät>:/etc/dropbear/authorized_keys


****Hinweise:****


Backup: Vor Modifikation von ssh-copy-id ein Backup erstellen.


Distribution: Die Funktionsweise von ssh-copy-id kann variieren.


**Das Problem wird in einer der nächsten Versionen von ssh-copy-id gefixt sein!**

# Freifunk-Installation auf dem Ubiquiti UniFi U6+ (U6-Plus)

Dieses Repository enthält ein Hilfsskript (`fwupdate.sh`) und eine Schritt-für-Schritt-Anleitung zur Installation der Freifunk-Software (Gluon) auf einem **Ubiquiti UniFi U6+ Access Point**.

## ⚠️ Wichtige Sicherheitshinweise

> **Achtung – Experimenteller Status:**  
> Die Firmware funktioniert, gilt jedoch noch als experimentell. 

* **Kein reguläres Update:** Diese Firmware kann **nicht** einfach als Update auf einen bereits konfigurierten Freifunk-Router eingespielt werden! Das System wird dadurch unbrauchbar.
* **Empfohlenes Vorgehen bei bestehender Installation:**
  1. Gerät mit `firstboot` zurücksetzen.
  2. Erst danach das neue Image per `sysupgrade` installieren.
  3. Nach dem Neustart erneut anmelden und zur Sicherheit nochmals `firstboot` gefolgt von einem `reboot` ausführen.
  4. Anschließend wie gewohnt die Konfiguration im Config-Mode durchführen.

# Freifunk-Installation auf dem Ubiquiti UniFi U6+ (U6-Plus)

Dieses Repository enthält ein Hilfsskript (`fwupdate.sh`) und eine Schritt-für-Schritt-Anleitung zur Installation der Freifunk-Software (Gluon) auf einem **Ubiquiti UniFi U6+ Access Point**.

---

## ⚠️ Wichtige Sicherheitshinweise

> **Achtung – Experimenteller Status:**  
> Die Firmware funktioniert, gilt jedoch noch als experimentell. 

* **Kein reguläres Update:** Diese Firmware kann **nicht** einfach als Update auf einen bereits konfigurierten Freifunk-Router eingespielt werden! Das System wird dadurch unbrauchbar.
* **Empfohlenes Vorgehen bei bestehender Installation:**
  1. Gerät mit `firstboot` zurücksetzen.
  2. Erst danach das neue Image per `sysupgrade` installieren.
  3. Nach dem Neustart erneut anmelden und zur Sicherheit nochmals `firstboot` gefolgt von einem `reboot` ausführen.
  4. Anschließend wie gewohnt die Konfiguration im Config-Mode durchführen.

## 2. Sonderfall: `ssh-copy-id` auf OpenWrt / UniFi

`ssh-copy-id` kopiert Schlüssel standardmäßig nach `~/.ssh/authorized_keys`. OpenWrt und UniFi-Systeme erwarten diese Datei jedoch unter `/etc/dropbear/authorized_keys`.

```bash
[ -f /etc/openwrt_release ] && ([ "$LOGNAME" = "root" ] || [ "$(id -u)" = "0" ]) &&
```

*Hinweis: In neueren Versionen von `ssh-copy-id` ist dieser Fix bereits integriert. Lege vor Modifikationen lokaler System-Skripte immer ein Backup an.*
